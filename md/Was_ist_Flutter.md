# Was ist Flutter?

## Definition

Flutter ist ein **Open-Source Framework** von Google für die Entwicklung von Anwendungen. Mit einem einzigen Quellcode können Sie Apps für mehrere Plattformen erstellen – mobile Geräte (iOS und Android), Web und Desktop.

**Kernaussage**: "Build beautiful, natively compiled, multi-platform applications from a single codebase"

---

## Grundlegende Merkmale

### 1. **Multi-Plattform Development**
- **Eine Codebasis** → Mehrere Plattformen
- Zielplattformen:
  - Mobil: iOS, Android
  - Web: Browser
  - Desktop: Windows, macOS, Linux
  - Embedded: IoT-Geräte

### 2. **Programmiersprache: Dart**
- Flutter basiert auf **Dart** – einer von Google optimierten Programmiersprache
- Dart wurde speziell für schnelle Apps entwickelt
- Dart ist:
  - **Objektorientiert** – Programmierung mit Klassen und Objekten (ähnlich wie Java, C++, Python). Ermöglicht strukturierte, wiederverwendbare Code-Organisation
  - **Typsicher** – Der Compiler überprüft bereits vor dem Ausführen, dass Variablen den richtigen Datentyp haben (z.B. nicht versehentlich Text in eine Zahlenvariable speichern). Das vermeidet viele Fehler zur Laufzeit.
  - **Garbage-collected** – Der Speicher wird automatisch verwaltet. Wenn Objekte nicht mehr verwendet werden, räumt der "Garbage Collector" automatisch auf und gibt Speicher frei. Der Entwickler muss sich nicht manuell um Speicherverwaltung kümmern (anders als in C++).

### 3. **Performance**
- Code wird **nativ kompiliert** (zu ARM oder Intel Machine Code sowie JavaScript)
- Führt zu schnellen Anwendungen auf allen Geräten
- Native Rendering für beste visueller Darstellung

### 4. **Hot Reload**
- Eine der **beliebtesten Features** für Entwickler
- Änderungen im Code sind **sofort sichtbar** (ohne Neustart)
- Drastisch kürzere Entwicklungszyklen

## Hauptmerkmale für die Entwicklung

### Widgets
- Flutter-Apps bestehen aus **Widgets** (wiederverwendbare UI-Komponenten)
- Material Design und Cupertino (iOS-Style) Widgets integriert

### Developer Experience
- Automatisierte Tests
- Developer Tooling
- Production-Quality Apps

### Stabilität & Zuverlässigkeit
- Unterstützt und gepflegt von Google
- Große globale Community
- Regelmäßige Updates

## Vorteile des Flutter Frameworks

✅ **Code Reuse** – Eine Codebasis für mehrere Plattformen  
✅ **Schnelle Entwicklung** – Hot Reload, schnelle Iteration  
✅ **Schöne UI** – Material Design und Cupertino Widgets  
✅ **Hohe Performance** – Native Kompilierung  
✅ **Große Community** – Viele Ressourcen und Pakete  
✅ **Open Source** – Kostenlos und transparent  
✅ **Google Support** – Aktiv entwickelt und gepflegt  

---

## Nachteile / Überlegungen

⚠️ **Dart-Kenntnisse** – Notwendig für Flutter-Entwicklung  
⚠️ **Größere App-Größe** – Gegenüber nativen Apps  
⚠️ **Jüngeres Framework** – Nicht so ausgereift wie einige Alternativen  
⚠️ **Native Funktionen** – Manchmal notwendig (über Platform Channels)
